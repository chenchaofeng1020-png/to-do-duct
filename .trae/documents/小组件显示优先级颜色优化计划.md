# 优化小组件：显示任务优先级颜色

用户希望在小组件中也能像主 App 一样，根据任务的优先级显示不同的颜色。

## 需求分析
目前小组件中的复选框图标颜色是硬编码的（未完成为灰色，已完成为绿色）。
我们需要：
1.  **复用优先级逻辑**：将 `TodoPriority` 枚举或其对应的颜色逻辑引入到 Widget 中。
2.  **更新 UI**：根据 `WidgetTask` 中的 `priorityRawValue` 动态设置复选框的颜色。

## 实施计划

1.  **引入 `TodoPriority`**：
    *   检查 `TodoPriority` 是否已在共享文件中定义，或者在 `DesignSystem` 中是否有对应颜色。如果 `TodoPriority` 是 `TodoItem.swift` 中的私有定义，我们需要将其移动到公共文件或在 Widget 中重新映射。
    *   为了简化，我们可以在 `WidgetView.swift` 中编写一个辅助函数，根据 `priorityRawValue` 返回对应的 `Color`。
        *   3 (High) -> Red
        *   2 (Medium) -> Orange
        *   1 (Low) -> Blue
        *   0 (None) -> Gray

2.  **修改 `WidgetView.swift`**：
    *   在 `MediumWidgetView` 和 `LargeWidgetView` 中。
    *   修改 `Image(systemName: ...)` 的 `.foregroundColor` 修饰符。
    *   逻辑：
        *   如果任务 **已完成** (`isDone`)：保持为 `DesignSystem.checkedColor` (绿色)。
        *   如果任务 **未完成**：根据优先级显示对应颜色。如果优先级为 0 (无)，则显示灰色。

## 颜色映射参考 (基于主 App 逻辑)
*   High (3): `.red`
*   Medium (2): `.orange`
*   Low (1): `.blue`
*   None (0): `.textTertiary` (Gray)

我将直接在 `WidgetView.swift` 中实现这个颜色映射逻辑。
