# 桌面小组件 (Widgets) 产品策划方案

## 1. 概述 (Overview)
**目标**：利用 iOS 桌面小组件 (WidgetKit) 提高用户活跃度，让用户无需打开 App 即可快速浏览今日任务进度，强化“治愈”与“陪伴”感。
**核心价值**：
*   **一瞥即知 (Glanceability)**：快速确认今天还有什么没做。
*   **情感连接 (Emotional Connection)**：通过鸭子的状态变化（开心/忙碌/睡觉）映射用户的任务状态。

## 2. 组件设计 (Widget Designs)

我们将首发支持两种最常用的尺寸：**小号 (Small)** 和 **中号 (Medium)**。

### 2.1 小号组件 (Small Widget) - "鸭鸭心情"
*   **定位**：情感化进度展示，类似于“电子宠物”。
*   **尺寸**：2x2 (169x169 pt)
*   **UI 布局**：
    *   **中心**：一只大大的 To-Do Duck 插画。
    *   **左上角**：当前日期（如 "12 周三"）。
    *   **底部**：环形或条形进度条，显示 "3/5"（已完成/总任务）。
*   **动态交互 (鸭子状态)**：
    *   **任务 < 30%**：鸭子在看书或思考（"加油鸭"）。
    *   **任务 > 60%**：鸭子在开心跳舞或比耶（"很棒鸭"）。
    *   **任务 100%**：鸭子戴着墨镜躺在沙滩椅上（"下班鸭"）。
    *   **无任务**：鸭子在睡觉（"休息鸭"）。
*   **点击行为**：打开 App 并跳转到“今天”的卡片页。

### 2.2 中号组件 (Medium Widget) - "今日清单"
*   **定位**：实用性为主，兼顾美观。
*   **尺寸**：2x4 (360x169 pt)
*   **UI 布局**：
    *   **左侧 (30%)**：
        *   当前日期（大字号）。
        *   一只小鸭子图标。
        *   进度概览文本（例如 "剩余 2 项"）。
    *   **右侧 (70%)**：
        *   展示今日的前 3-4 条未完成任务。
        *   每条任务显示：复选框 (视觉元素，不可点) + 标题 + 优先级颜色标记。
        *   如果任务超过 4 条，底部显示 "...还有 X 项"。
        *   如果所有任务已完成，显示“🎉 今日任务全部搞定！”文案。
*   **点击行为**：
    *   点击任务区域：打开 App 并滚动到该任务位置（Deep Link）。
    *   点击左侧区域：打开 App 首页。

## 3. 交互逻辑 (Interaction)

### 3.1 刷新机制
*   **自动刷新**：
    *   用户在主 App 中添加/完成/修改任务后，强制刷新 Widget。
    *   系统智能刷新（System Intelligence）：iOS 会根据用户使用习惯定期刷新。
*   **过夜刷新**：每天凌晨 0 点，自动更新为新的一天（显示空白或新生成的卡片）。

### 3.2 空状态处理 (Empty State)
*   如果今天还没有创建卡片：显示“新的一天开始啦！”引导文案，鸭子举着“+”号牌子。
*   点击直接触发“创建今日卡片”的动作。

## 4. 技术实现要点 (Technical Specs)

### 4.1 框架选择
*   使用 **WidgetKit** + **SwiftUI**。
*   数据共享：使用 **App Group** (`group.com.yourname.tododuck`) 实现主 App 与 Widget 之间的数据共享。由于使用了 **SwiftData**，需要配置共享的 `ModelContainer`。

### 4.2 数据模型
*   创建一个轻量级的 `WidgetEntry` 模型，只包含渲染所需的数据，避免在 Widget 中进行复杂的数据库查询。
    ```swift
    struct SimpleEntry: TimelineEntry {
        let date: Date
        let completedCount: Int
        let totalCount: Int
        let topTasks: [String] // 仅存标题
        let duckStatus: DuckStatus // 枚举：working, happy, chill
    }
    ```

### 4.3 互动性 (iOS 17+)
*   虽然 iOS 17 支持 Widget 直接交互（如直接点击打钩），但考虑到开发成本和数据一致性风险，**第一版建议暂不支持 Widget 上直接打钩**，仅作为展示和跳转入口。待功能稳定后，可在 v1.2 版本加入 `AppIntent` 实现交互。

## 5. 开发排期 (Roadmap)

*   **Week 1**: 
    *   配置 App Group。
    *   实现 SwiftData 在 Widget Extension 中的读取。
    *   开发“小号组件”UI。
*   **Week 2**: 
    *   开发“中号组件”UI。
    *   实现鸭子状态根据完成率自动切换的逻辑。
    *   联调跳转协议 (Deep Link)。
*   **Week 3**: 
    *   测试数据同步的及时性。
    *   优化空状态和夜间模式适配。
